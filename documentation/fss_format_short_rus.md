# Формат FSS (File Structure Snapshot) "FSS-YAML"

## Описание

- **Кодировка**: UTF-8
- **Формат**: YAML

FSS — формат для представления структуры файловой системы в виде плоского YAML-словаря. Каждый ключ — это путь к файлу или директории, каждое значение — словарь с метаданными.

## Структура

```yaml
"путь/к/объекту":
  атрибут1: значение1
  атрибут2: значение2
```

**Пример файла:**
```yaml
"document.txt":
  type: file
  size: 1024
  md5: "5d41402abc4b2a76b9719d911017c592"
  ctime: "2024-03-15_10:30:00Z"
  mtime: "2024-03-20_14:45:00Z"
```

**Пример директории:**
```yaml
"photos":
  type: dir
  ctime: "2024-01-10_12:00:00Z"
  mtime: "2024-03-15_16:30:00Z"
```

**Пример с вложенными путями:**
```yaml
"docs":
  type: dir
  ctime: "2024-01-01_10:00:00Z"
  mtime: "2024-02-15_14:00:00Z"

"docs/readme.md":
  type: file
  size: 512
  md5: "098f6bcd4621d373cade4e832627b4f6"
  ctime: "2024-01-05_11:00:00Z"
  mtime: "2024-02-15_14:00:00Z"

"docs/images/logo.png":
  type: file
  size: 32768
  md5: "482c811da5d5b4bc6d497ffa98491e38"
  ctime: "2024-01-05_11:30:00Z"
  mtime: "2024-01-05_11:30:00Z"
```

## Ключи

- **Относительные пути** от базовой директории
- **Разделитель**: `/` (POSIX-стиль)
- **Регистр**: сохраняется оригинальный

## Значения

Словарь с метаданными объекта файловой системы.

- `type` — тип объекта: `file`, `dir`, `unknown`, `error`, `hardcoded_skip`
- `ctime` — время создания (формат: `YYYY-MM-DD_HH:MM:SSZ`, UTC)
- `size` — размер в байтах (целое число)
- `md5` — MD5-хеш содержимого (32 hex символа)
- `zeros` — `true` если файл состоит только из нулевых байтов
- `error` — `true` или описание ошибки чтения
- `symlink` — `true` если объект является символической ссылкой

## Особенности

- **Плоская структура**: все объекты на одном уровне словаря, без вложенности
- **Масштабируемость**: поддержка частичной загрузки (потоковая загрузка данных) и обработки больших файлов
- **Эффективное использование памяти**: возможность частичной загрузки данных (потоковая загрузка данных) - в памяти находится информация только про 1 запись из файла FSS-YAML.

**Недостатки иерархической структуры**:
- ❌ Сложный доступ к вложенным файлам
- ❌ Проблемы с глубокой вложенностью в YAML
- ❌ Невозможность частичной обработки
- ❌ Высокое потребление памяти
- ❌ Высокий расход диского места изза хранения отступов
- ❌ Сложно редактировать человеку в текстовом редакторе - легко сделать ошибки при редактировании

**Преимущества плоской структуры**:
- ✅ Простой поиск по полному пути (O(1))
- ✅ Легко редактировать вручную
- ✅ Отсутствие проблем с отступами YAML
- ✅ Частичная загрузка больших файлов (streaming)
- ✅ Простое слияние нескольких файлов FSS YAML в один файл
- ✅ Легко редактировать человеку в текстовом редакторе
- ✅ Можно написать упрощенный парсер для чтения файла FSS YAML



# Атрибуты

## `ctime`, `mtime` - Временные метки

Все временные метки сохраняются в формате ISO8601 с модификацией:
- Формат: `YYYY-MM-DD_HH:MM:SSZ`
- Разделитель между датой и временем: подчёркивание `_` (вместо стандартного `T`)
- Часовой пояс: всегда UTC (обозначается суффиксом `Z`)
- Точность: секунды (миллисекунды отбрасываются)


### `md5` - MD5-хеш содержимого

- **Алгоритм**: MD5 (128 бит = 32 hex символа)
- **Размер блока чтения**: 65536 байт (64 КБ)
- **Файлы нулевого размера**: атрибут `md5` отсутствует
- **Ошибка чтения**: атрибут `md5` отсутствует, устанавливается `error: true`

**Оптимизация пересчёта**:
MD5 пересчитывается только при изменении:
- Размера файла (`size`)
- Времени создания (`ctime`)
- Времени модификации (`mtime`)
