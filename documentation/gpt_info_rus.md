# About (rus)

File struct saver. `fssaver`. Программа для сохранения структуры файлов в файл.

## Описание

Программа "fssaver" (File Structure Saver) — это набор скриптов для сохранения и управления структурой файловой системы.

**Позволяет:**

- **Сохранять структуру файловой** системы в один "fss-файл", содержащий всю структуру выбранного каталога.
- **Объединять множество "fss-файлов" в один**. Скрипт рекурсивно собирает все "fss-файлы" и собирает один "fss-файл" содержащий всю информацию из всех файлов.
- **Сравнивать два "fss-файла"**, и определять добавленные, удалённые, изменённые или перемещённые файлы между двумя структурами.

Такой подход позволяет:

- Отслеживать изменения в файловой системе.
- Синхронизировать данные между разными хранилищами.
- Проверять целостность файлов.
- Выполнять резервное копирование на основе сохранённых структур.

Фичи:

- **кэширование** - программа создаёт "fss-файлы" в каждом каталоге. При повторном запуске происходит обновление этих файлов с учетом изменений файловой системы.
- **YAML-файлы** - простой формат, удобный для редактирования.
- **простые скрипты** - удобно для изменения работы программы.

# Перечень скриптов:

## `fss_save.py`
Сохраняет структуру файловой системы в виде YAML-файлов.
Файл с структурой директории, создается в самой директории (в текущей директории).
Программа рекурсивно обходит директории и создает файлы с структурой (в каждой директории по одному файлу).
Имя файлов всегда ".index_hash.yaml".

Также скрипт может обновлять информацию содержащуюся в уже созданных файлах ".index_hash.yaml". Например их хеши и временные метки, или добавлять в структуру новые файлы (или удалять их).

## `fss_merge.py`
Этот скрипт объединяет структуры файлов из различных директорий в один YAML-файл.
Скрипт рекурсивно обходит директории, собирая информацию из файлов ".index_hash.yaml" и сохраняет эту информацию в один большой YAML-файл.

## `fss_compare.py`
Сравнивает две структуры файлов, представленные в виде YAML-файлов.
Скрипт определяет изменения, такие как перемещение, удаление, изменение или добавление файлов, и выводит результаты сравнения.

## `fss_yaml_clean.py`
Этот скрипт удаляет указанные атрибуты из элементов YAML-файла, опционально фильтруя элементы по их типу. Это полезно для очистки данных и удаления ненужной информации из структуры файлов.

## `fss_to_diskdir.py`
Этот скрипт преобразует структуру файлов из YAML-файла в текстовый формат, совместимый с плагинами DiskDir для файловых менеджеров, таких как Double Commander или Total Commander.

## `fss_utils.py`
Этот модуль содержит вспомогательные функции для работы с YAML-файлами, такие как загрузка и сохранение данных, а также другие утилиты для обработки файловой системы. Он используется другими скриптами для унификации и упрощения работы с данными.

# Формат файла

## Общая структура

Файл `.index_hash.yaml` представляет собой YAML-документ, содержащий словарь (dictionary), где:
- **Ключи**: относительные пути файлов и директорий
- **Значения**: метаданные о файлах и директориях


## Пример структуры для файла после выполнения скрипта `fss_save.py`

В этом файле содержится информация о файлах (и директориях) в текущей директории.

```yaml
"example_file.txt":
  ctime: 2023-05-12_19:44:54Z
  md5: fe70e52cec7d052b332cd22151a70515
  mtime: 2014-11-28_22:50:06Z
  size: 357907
  type: file
"example_file2.txt":
  ctime: 2023-05-12_18:40:50Z
  md5: 1234567890abcdef0000000000000000
  mtime: 2014-11-28_22:50:06Z
  size: 123456
  type: file
"images":
  type: dir
  ctime: "2024-03-19_15:45:00Z"
```

Каталог "images":

```yaml
"photos":
  type: dir
  ctime: "2024-03-19_15:45:00Z"
  mtime: "2024-03-20_09:15:00Z"
```

## Атрибуты записей

### Общие атрибуты (для файлов и директорий)
- `type`: тип записи
  - `file`: файл
  - `dir`: директория
  - `unknown`: неизвестный тип
  - `error`: ошибка при чтении
  - `hardcoded_skip`: пропущенный элемент
- `ctime`: время создания в формате ISO8601 GMT
- `mtime`: время модификации в формате ISO8601 GMT
- `symlink`: (опционально) `true` если это символическая ссылка
- `mod`: (опционально, Linux) строка с правами доступа, три цифры в восьмеричном формате, пример: `755`, `644`
- `own`: (опционально, Linux) строка с владельцем в формате `user:group`, пример: `root:root`

### Атрибуты файлов
- `size`: размер файла в байтах
- `md5`: MD5-хеш содержимого файла
- `zeros`: (опционально) `true` если файл содержит только нули
- `error`: (опционально) `true` если произошла ошибка при чтении файла

## Преимущества "плоской" структуры
- Легкий поиск по пути
- Возможность частичной обработки файла
- Эффективное использование памяти
- Простое объединение и редактирование файлов
- Отсутствие проблем с отступами
- Возможность работы с большими файлами


# Подробности работы скриптов

## fss_merge.py
1. Читает содержимое найденного файла и добавляет в словарь
2. Рекурсивно обрабатывает все поддиректории
3. Собирает информацию из всех найденных файлов
4. Сохраняет информацию в один большой YAML-файл

### Принцип работы
1. Скрипт начинает работу с указанной директории и ищет в ней файл `.index_hash.yaml`
2. Читает содержимое найденного файла и добавляет в словарь
3. Рекурсивно обрабатывает все поддиректории
4. Собирает информацию из всех найденных файлов `.index_hash.yaml`
5. Сохраняет информацию в один большой YAML-файл
6. Все пути в выходном файле преобразуются в относительные, относительно стартовой директории

### Выходной файл
- Формат выходного файла идентичен формату `.index_hash.yaml`, но содержит записи из всех объединенных файлов
- Все пути в выходном файле являются относительными от корневой директории
- Структура остается плоской - все файлы и директории находятся в одном словаре (без вложенных словарей)
- Сохраняются все метаданные из исходных файлов (размеры, хеши, временные метки)
- При наличии дубликатов путей (например, при слиянии одинаковых файлов из разных директорий) сохраняется последняя встреченная версия
- Файл можно частично парсить, обрабатывать и редактировать, так как он имеет плоскую структуру
